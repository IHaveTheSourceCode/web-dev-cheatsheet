WebDev Cheatsheet

Closure

Passed functions that are called outside of their scope retain acces to their original scope.

Factory Function

Function that creates object and returns it. Uses camelCase notation.

Constructor Function

Uses "this" keyword to set object's properties and methods, and "new" to initialize new object. Uses PascalNamingConvention.

Higher Order Function

Function that takes another function as an argument, or function that returns another function.

Impure Function

Uses variables declared outside of the scope, which gives a risk of mutatning them.

Pure Function

Relies only on locally declared variables, thereby removing risk of mutating variables.

Temporal Dead Zone

Is created above every variable declared with let or const. Blocks acces to the declared variable to everything above it.

Dynamic typing
vs
Static typing

Dynamicaly typed language doesn't need to know types of defined variables before compiling, which is opposite to static typed language.

Strict Mode

Throws errors when assigning value to non existing variable, object or property. Does not allow to delete variable, object, function or duplicate a parameter.

Minification

Minimazing code by deleting characters that are not interpreted by the browser e.g., whitespaces.

Buffering

Transfering data from memory with a relatively long acces time (e.g.: website server) to the memory with shorter accest time (local memory).

"Expires" and "Cache-Control" Captions

"Expires" tells the browser when will the file expire, and will have to be downloaded again. "Cache-Control" informs what time from download file should be downloaded again.

Application Programming Interface

Defines how different programs communicate between each other.

REST

Representational State Transfer - is a design concept for managing state information.

Array Methods

pop(), push(), shift(), unshift(), splice(), slice(), arr.join(), Array.from(), arr.concat(), arr.length, arr.map(), arr.reduce(), arr.filter().

Values

Primitive values: number, boolean, string, undefined, null, bigInt, symbol.
Other type values: objects, functions.

Destructuring

JavaScript expression that makes it possible to unpack values from arrays, or properties from objects, into distinct variables.

Managing "this" keyword reference

().call(), ().apply(), ().bind()

Variable's Scopes

Variables defined with "var" are function scoped, while "let" and "const" are block scoped.

Modules

Modules are created as IIFE, which return an object with its selected parameters and methods.

Defining Private Methods

Private methods start with "_" e.g. _privateMethod

Revealing Module Pattern

One of the most popular ways of creating modules. Uses the return statement to return an object literal that "reveals" only the methods or proporties we want to be publicly avaible.

Composition over Inheritance

Composition defines object as the sum of its parts, thereby is better than inheritance from one class to another, because it doesn't give a risk of breaking code when changing parent class.

Methods On Variables

Although code like "hi".toUpperCase() makes "hi" look like an object, this is nothing but illusion. Javascript creates a temporary object when you do this, and then immediately discards it.

Function expression vs Function declaration

Function expression declares function and assigns it to variable. Function declaration does only create function without assigning it to variable.

Encapsulation

One of the basic concepts of the OOP, assumes that it's better to hide certain parameters or methods, so they won't be used outside of the parent object.

Prototypal Inheritance

Prototypal Inheritance is a method by which an object can inherit the properties and methods from another object.

Getters
and
Setters

Object's methods that get or set values of an object.

Classes

Javascript syntax, that does the same thing as the object constructors and prototypes.

Function binding

Uses "bind" method to bind this keyword to object, and/or set values on functions.

Mix-ins

Templates for classes, which allow us to take superclass as input and sublass extending that superclass as output.
